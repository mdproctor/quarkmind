QA Harness — Phases 0 & 1

Dummy plugins, real SC2 integration — full pipeline exercisable before any real intelligence exists

👁 Observe
Verify what the bot sees from SC2 is reaching the CaseFile correctly.
REST endpoint: GET /sc2/casefile
Live snapshot of CaseFile state — see exactly what the bot sees at any moment
REST endpoint: GET /sc2/casefile/stream
SSE stream of CaseFile writes — watch observations arrive in real time
REST endpoint: GET /sc2/plugins
Which plugins are active at each level, activation counts, last-fired timestamp
Structured logs
Every CaseFile write and TaskDefinition activation logged with game time
🎯 Direct
Verify that intents from plugins actually reach SC2 and produce the expected unit behaviour.
REST endpoint: POST /sc2/intent
Inject a typed intent directly into IntentQueue — e.g. move a probe, attack a point — without any plugin logic
REST endpoint: GET /sc2/intents/pending
See what intents are queued but not yet dispatched — verify pipeline isn't stalling
REST endpoint: GET /sc2/intents/dispatched
Ring-buffer of last N dispatched intents — confirm commands are reaching SC2
Frame counter & tick rate
Verify SC2 frame loop is running at expected pace — detect stalls or overruns
⚡ Interact
Trigger scenarios against a live SC2 game to verify the integration handles real game events correctly.
SC2 Debug API
ocraft-s2client exposes SC2's debug interface — spawn units, set resources, create attacks — for scripted scenarios against a live game
REST endpoint: POST /sc2/debug/scenario
Trigger a named debug scenario (e.g. "spawn_enemy_attack", "set_minerals_500") — verify CaseFile reacts correctly
REST endpoint: POST /sc2/casefile/{key}
Manually write a CaseFile key to test plugin reactions — simulate what a real plugin would write
Step mode support
SC2 API supports step mode (advance N frames on demand) — enables deterministic integration tests without real-time timing pressure
Unit test mode — no SC2 needed
SimulatedGameObserver (%test profile) injects scripted GameState snapshots. Tests plugin logic in isolation, fast feedback loop. No SC2 install required.
Integration test mode — live SC2
Real SC2 in step mode + Debug API + REST QA endpoints. Full pipeline exercised: SC2 → GameObserver → CaseFile → plugins → IntentQueue → SC2 commands. Scenarios triggered via REST.
All QA endpoints are %dev/%test profile only — stripped from production builds. No security surface in deployed bot.
